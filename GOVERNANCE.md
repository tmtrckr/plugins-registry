# Governance Model

## Roles

### Core Maintainers

- Final approval for critical changes
- Security incident response
- Governance decisions
- **Current:** (Add maintainer GitHub usernames here, e.g. @maintainer1, @maintainer2)

### Trusted Contributors

- Review and merge routine PRs
- Triage issues
- Community support
- **How to become:** 10+ merged PRs and nomination by an existing maintainer

### Contributors

- Submit PRs
- Report issues
- Participate in discussions

## Decision Making

- **Routine changes:** Single maintainer approval
- **Significant changes:** 2+ maintainer approval
- **Governance changes:** Consensus of all maintainers
- **Emergency security:** Single maintainer can act immediately

## Adding Maintainers

1. Nomination by existing maintainer
2. 2-week public comment period
3. Unanimous approval required

## Conflict Resolution

1. Discussion in GitHub issue or PR
2. If unresolved → maintainer discussion
3. If still unresolved → vote (majority wins)
